# Jerry Food 微信小程序基础框架

这是一个微信小程序的基础框架示例，用于演示如何在本地部署和访问微信小程序。

## 项目结构

```
jerry-food/
├── app.json          # 小程序配置文件
├── app.js            # 小程序入口文件
├── app.wxss          # 小程序全局样式
├── sitemap.json      # 小程序站点地图
├── pages/
│   ├── index/        # 首页
│   │   ├── index.wxml
│   │   ├── index.js
│   │   └── index.wxss
│   └── about/        # 关于页面
│       ├── about.wxml
│       ├── about.js
│       └── about.wxss
├── package.json      # 服务器依赖配置
├── server.js         # 本地服务器
└── README.md         # 项目说明
```

## 功能特点

- ✅ 基础的微信小程序结构
- ✅ 首页和关于页面
- ✅ 页面跳转功能
- ✅ 本地服务器API接口
- ✅ 服务器连接测试

## 部署步骤

### 1. 安装Node.js

如果你的电脑还没有安装Node.js，请先下载并安装：

- 访问 [Node.js官网](https://nodejs.org/zh-cn/)
- 下载并安装适合你系统的版本
- 安装完成后，打开命令提示符，运行 `node -v` 确认安装成功

### 2. 安装服务器依赖

在项目根目录运行：

```bash
npm install
```

### 3. 启动本地服务器

```bash
npm start
```

服务器将运行在 `http://localhost:3000`

### 4. 在微信开发者工具中导入项目

1. 打开微信开发者工具
2. 点击「导入项目」
3. 选择项目根目录 `jerry-food`
4. 填写AppID（如果没有，可以选择「测试号」）
5. 点击「导入」

### 5. 测试本地服务器连接

1. 在小程序首页，点击「测试本地服务器」按钮
2. 如果服务器正常运行，会显示「本地服务器连接成功！」
3. 如果连接失败，请检查服务器是否已启动

## 本地服务器API接口

- `GET /api/test` - 测试接口，返回服务器连接状态
- `GET /api/health` - 健康检查接口，返回服务器运行状态

## 注意事项

1. 确保本地服务器已启动
2. 微信小程序开发工具中需要开启「不校验合法域名」选项（在设置 -> 项目设置中）
3. 手机访问时，需要确保手机和电脑在同一局域网内
4. 手机访问时，需要将小程序中的 `localhost` 替换为电脑的本地IP地址

## 开发指南

### 添加新页面

1. 在 `app.json` 的 `pages` 数组中添加新页面路径
2. 在 `pages` 目录下创建对应文件夹和文件
3. 编写页面的 wxml、js 和 wxss 文件

### 扩展服务器API

在 `server.js` 文件中添加新的路由和处理函数

## 技术栈

- 微信小程序原生开发
- Express.js 本地服务器
- Node.js 运行环境
